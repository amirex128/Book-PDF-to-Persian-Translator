انواع windows

هنگامی که شما می‌دانید که چگونه باید timestamp یک رویداد را تعیین کنید، گام بعدی این است که تصمیم بگیرید که چگونه باید windows ها را در دوره‌های زمانی تعریف کنید. سپس می‌توان از window برای تجمیع‌ها استفاده کرد، به عنوان مثال برای شمارش رویدادها، یا برای محاسبه‌ی میانگین مقادیر در داخل window. چندین نوع از windows در استفاده‌ی رایج هستند [79, 83]:

  • Tumbling window

    • یک tumbling window دارای طول ثابتی است، و هر رویداد دقیقاً به یک window تعلق دارد. به عنوان مثال، اگر شما یک tumbling window یک دقیقه‌ای دارید، تمام رویدادها با timestamps بین 10:03:00 و 10:03:59 در یک window گروه‌بندی می‌شوند، رویدادها بین 10:04:00 و 10:04:59 در window بعدی، و به همین ترتیب. شما می‌توانید یک tumbling window یک دقیقه‌ای را با گرفتن هر timestamp رویداد و گرد کردن آن به نزدیک‌ترین دقیقه برای تعیین window ای که به آن تعلق دارد، پیاده‌سازی کنید.

  • Hopping window

    • یک hopping window نیز دارای طول ثابتی است، اما به windows اجازه می‌دهد که برای ارائه مقداری smoothing همپوشانی داشته باشند. به عنوان مثال، یک window 5 دقیقه‌ای با یک اندازه hop از 1 دقیقه شامل رویدادها بین 10:03:00 و 10:07:59 خواهد بود، سپس window بعدی رویدادها را بین 10:04:00 و 10:08:59 پوشش می‌دهد، و به همین ترتیب. شما می‌توانید این hopping window را با محاسبه ابتدا tumbling windows یک دقیقه‌ای، و سپس تجمیع بر روی چندین windows مجاور، پیاده‌سازی کنید.

  • Sliding window

    • یک sliding window شامل تمام رویدادهایی است که در یک فاصله‌ی زمانی رخ می‌دهند. به عنوان مثال، یک sliding window 5 دقیقه‌ای، رویدادها را در 10:03:39 و 10:08:12 پوشش می‌دهد، زیرا آن‌ها کمتر از 5 دقیقه از هم فاصله دارند (توجه داشته باشید که windowsهای 5 دقیقه‌ای tumbling و hopping این دو رویداد را در یک window قرار نمی‌دادند، زیرا آن‌ها از boundaries ثابت استفاده می‌کنند). یک sliding window را می‌توان با نگه داشتن یک buffer از رویدادها که بر اساس زمان مرتب شده‌اند و حذف رویدادهای قدیمی هنگامی‌که از window منقضی می‌شوند، پیاده‌سازی کرد.

  • Session window

    • بر خلاف سایر انواع window، یک session window هیچ مدت زمان ثابتی ندارد. در عوض، با گروه‌بندی تمام رویدادها برای یک کاربر یکسان که در زمان با هم رخ می‌دهند، تعریف می‌شود، و window زمانی پایان می‌یابد که کاربر مدتی غیرفعال بوده است (به عنوان مثال، اگر هیچ رویدادی برای 30 دقیقه وجود نداشته باشد). Sessionization یک نیاز رایج برای تجزیه و تحلیل وب‌سایت است (نگاه کنید به "GROUP BY" در صفحه 406).
