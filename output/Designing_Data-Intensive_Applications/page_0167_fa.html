PART II

Distributed Data

For a successful technology, reality must take precedence over public relations, for nature cannot be fooled. —Richard Feynman, Rogers Commission Report (1986)

در قسمت I این کتاب، ما جنبه‌هایی از data systems را که هنگام ذخیره داده‌ها بر روی یک machine واحد اعمال می‌شوند، مورد بحث قرار دادیم. اکنون، در قسمت II، ما یک level را بالا می‌بریم و می‌پرسیم: اگر چندین machine در storage و retrieval از data دخیل باشند، چه اتفاقی می‌افتد؟

دلایل مختلفی وجود دارد که چرا شما ممکن است بخواهید یک database را در سراسر multiple machines توزیع کنید:

  • Scalability
  • اگر حجم داده‌های شما، read load یا write load بزرگتر از یک machine واحد شود، شما به طور بالقوه می‌توانید load را در سراسر multiple machines پخش کنید.
  • Fault tolerance/high availability
  • If your application نیاز دارد که به کار خود ادامه دهد، حتی اگر یک machine (یا چندین machines، یا network یا یک datacenter کامل) از کار بیفتد، شما می‌توانید از multiple machines برای ارائه redundancy استفاده کنید. هنگامی که یکی fail می‌شود، دیگری می‌تواند کنترل را در دست بگیرد.
  • Latency
  • اگر شما users در سراسر جهان دارید، شما ممکن است بخواهید servers را در locations مختلف در سراسر جهان داشته باشید تا هر user بتواند از یک datacenter که از نظر جغرافیایی به آن‌ها نزدیک است، خدمات دریافت کند. این باعث می‌شود که users مجبور نباشند منتظر network pack‐ets بمانند تا در سراسر جهان سفر کنند.
