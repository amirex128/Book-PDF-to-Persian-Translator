CHAPTER 1

Reliable, Scalable, and Maintainable Applications

اینترنت آن‌قدر خوب انجام شد که اکثر مردم آن را به‌عنوان یک منبع طبیعی مانند اقیانوس آرام در نظر می‌گیرند، نه چیزی که ساخته دست بشر باشد. آخرین باری که یک فناوری با چنین مقیاسی بدون خطا بود، کی بود؟

—Alan Kay, in interview with Dr Dobb’s Journal (2012)

امروزه بسیاری از applications data-intensive هستند، برخلاف compute-intensive. قدرت CPU خام به‌ندرت یک عامل محدودکننده برای این applications است—مشکلات بزرگ‌تر معمولاً حجم data، پیچیدگی data و سرعتی است که در آن در حال تغییر است.

یک data-intensive application معمولاً از building blocks استاندارد ساخته می‌شود که عملکرد موردنیاز را ارائه می‌دهند. به‌عنوان‌مثال، بسیاری از applications به موارد زیر نیاز دارند:

  • ذخیره data به‌طوری‌که آن‌ها یا یک application دیگر، بتوانند بعداً آن را دوباره پیدا کنند (databases)
  • نتیجه یک عملیات پرهزینه را به خاطر بسپارید تا سرعت خواندن را افزایش دهید (caches)
  • به کاربران اجازه دهید data را بر اساس کلمه کلیدی جستجو کنند یا آن را به روش‌های مختلف فیلتر کنند (search indexes)
  • یک پیام را به فرآیند دیگری ارسال کنید تا به‌صورت asynchronously رسیدگی شود (پردازش جریانی)
  • به‌طور دوره‌ای مقدار زیادی از data انباشته شده را خرد کنید (پردازش دسته‌ای)

اگر این به‌طور دردناکی واضح به نظر می‌رسد، فقط به این دلیل است که این سیستم‌های data یک انتزاع بسیار موفق هستند: ما همیشه از آن‌ها استفاده می‌کنیم، بدون اینکه زیاد فکر کنیم. هنگام ساختن یک application، اکثر مهندسان خواب نوشتن یک موتور ذخیره‌سازی data جدید را از ابتدا نمی‌بینند، زیرا databases یک ابزار کاملاً خوب برای این کار هستند.
