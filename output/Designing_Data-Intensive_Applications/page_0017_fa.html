این کتاب برای مهندسان software، معماران software و مدیران فنی است که عاشق کدنویسی هستند. این امر به‌ویژه در صورتی مرتبط است که نیاز به تصمیم‌گیری در مورد architecture سیستم‌هایی دارید که روی آن‌ها کار می‌کنید—به‌عنوان‌مثال، اگر نیاز به انتخاب ابزارهایی برای حل یک مشکل خاص دارید و می‌خواهید بدانید که چگونه بهترین روش را برای اعمال آن‌ها به‌کار برید. اما حتی اگر در مورد ابزارهای خود هیچ انتخابی ندارید، این کتاب به شما کمک می‌کند تا نقاط قوت و ضعف آن‌ها را بهتر درک کنید.

شما باید مقداری تجربه در ساخت web-based applications یا network services داشته باشید و باید با relational databases و SQL آشنا باشید. هر non-relational databases و سایر ابزارهای مرتبط با data که می‌شناسید یک امتیاز است، اما ضروری نیست. درک کلی پروتکل‌های شبکه‌ای رایج مانند TCP و HTTP مفید است. انتخاب زبان برنامه‌نویسی یا فریم‌ورک شما هیچ تفاوتی برای این کتاب ایجاد نمی‌کند.

اگر هر یک از موارد زیر برای شما صدق می‌کند، این کتاب را ارزشمند خواهید یافت:

  • شما می‌خواهید یاد بگیرید که چگونه سیستم‌های data را مقیاس‌پذیر کنید، به‌عنوان‌مثال، برای پشتیبانی از برنامه‌های web یا mobile با میلیون‌ها کاربر.
  • شما باید applications را در دسترس قرار دهید (به حداقل رساندن زمان خرابی) و از نظر عملیاتی قوی.
  • شما به دنبال راه‌هایی برای آسان‌تر کردن نگهداری سیستم‌ها در درازمدت هستید، حتی با رشد و تغییر الزامات و فناوری‌ها.
  • شما یک کنجکاوی طبیعی برای نحوه عملکرد چیزها دارید و می‌خواهید بدانید در داخل وب‌سایت‌های بزرگ و online services چه می‌گذرد. این کتاب، درون databases و سیستم‌های پردازش data مختلف را تجزیه می‌کند و کاوش در تفکر روشن‌بینانه‌ای که به طراحی آن‌ها منتهی شده است، بسیار سرگرم‌کننده است.

گاهی اوقات، هنگام بحث در مورد سیستم‌های data مقیاس‌پذیر، مردم نظراتی در این راستا ارائه می‌دهند: "شما Google یا Amazon نیستید. نگران مقیاس نباشید و فقط از یک relational database استفاده کنید." در این گفته حقیقت وجود دارد: ساختن برای مقیاسی که به آن نیاز ندارید، تلاش هدر رفته است و ممکن است شما را در یک طراحی غیرقابل انعطاف قفل کند. در واقع، این یک نوع بهینه‌سازی زودرس است. با این حال، انتخاب ابزار مناسب برای این کار نیز مهم است و فناوری‌های مختلف هرکدام نقاط قوت و ضعف خاص خود را دارند. همان‌طور که خواهیم دید، relational databases مهم هستند اما آخرین کلمه در مورد برخورد با data نیستند.

Scope of This Book

این کتاب تلاش نمی‌کند دستورالعمل‌های دقیقی در مورد نحوه نصب یا استفاده از بسته‌های software یا APIs خاص ارائه دهد، زیرا در حال حاضر مستندات زیادی برای آن موارد وجود دارد. در عوض ما در مورد اصول و مبادلات مختلفی که برای سیستم‌های data اساسی هستند بحث می‌کنیم و تصمیمات طراحی مختلفی را که توسط محصولات مختلف گرفته شده است، بررسی می‌کنیم.
