جدول 13.9  Shared-Data Pattern Solution

Overview ارتباط بین data accessors، توسط یک shared-data store واسطه می‌شود. کنترل ممکن است توسط data accessors یا data store آغاز شود. داده‌ها توسط data store، پایدار می‌شوند.
Elements Shared-data store. نگرانی‌ها شامل انواع داده‌های ذخیره‌شده، ویژگی‌های داده‌ها محور performance، توزیع داده‌ها و تعداد accessors مجاز است.
Data accessor component.
Data reading and writing connector. یک انتخاب مهم در اینجا این است که آیا connector، transactional است یا خیر، و همچنین زبان خواندن/نوشتن، پروتکل‌ها و semantics.
Relations رابطه پیوست، components accessors را با data stores مربوطه مرتبط می‌کند.
Constraints Data accessors با data store(s) تعامل دارند.
Weaknesses The shared-data store ممکن است یک گلوگاه performance باشد.
The shared-data store ممکن است یک نقطه شکست واحد باشد.
تولیدکنندگان و مصرف‌کنندگان داده‌ها ممکن است به‌طور محکم coupled شوند.
