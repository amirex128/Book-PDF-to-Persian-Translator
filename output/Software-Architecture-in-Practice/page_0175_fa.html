9.3 A Design Checklist for Security

جدول 9.2 یک checklist برای پشتیبانی از فرآیند طراحی و تحلیل برای security است.

Category Checklist
Allocation of Responsibilities تعیین کنید که کدام system responsibilities باید ایمن باشند. برای هر یک از این responsibilities، اطمینان حاصل کنید که responsibilities اضافی برای انجام موارد زیر تخصیص داده شده‌اند:
  • شناسایی actor
  • Authenticate کردن actor
  • Authorize کردن actors
  • اعطا یا رد دسترسی به داده‌ها یا services
  • ثبت تلاش‌ها برای دسترسی یا اصلاح داده‌ها یا services
  • رمزنگاری داده‌ها
  • تشخیص کاهش availability برای منابع یا services و اطلاع‌رسانی به پرسنل مربوطه و محدود کردن دسترسی
  • بازیابی از یک حمله
  • تأیید checksums و مقادیر hash
