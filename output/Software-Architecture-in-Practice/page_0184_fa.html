10.1  Testability General Scenario

جدول 10.1 عناصر سناریوی کلی را که testability را مشخص می‌کند، فهرست می‌کند.

Portion of Scenario Possible Values
Source Unit testers, integration testers, system testers, acceptance testers, کاربران نهایی، چه اجرای tests به صورت دستی یا با استفاده از ابزارهای automated testing
Stimulus مجموعه‌ای از tests به دلیل تکمیل یک coding increment مانند یک لایه یا service class، یکپارچه‌سازی تکمیل‌شده یک زیرسیستم، پیاده‌سازی کامل کل سیستم، یا تحویل سیستم به مشتری اجرا می‌شود.
Environment زمان طراحی، زمان توسعه، زمان کامپایل، زمان یکپارچه‌سازی، زمان استقرار، زمان اجرا
Artifacts بخشی از سیستم که در حال test است
Response یک یا چند مورد از موارد زیر: اجرای مجموعه test و ثبت نتایج، ثبت فعالیتی که منجر به خطا شده است، کنترل و monitor کردن state سیستم
Response Measure یک یا چند مورد از موارد زیر: تلاش برای یافتن یک خطا یا کلاس خطاها، تلاش برای دستیابی به یک درصد معین از پوشش فضای state، احتمال آشکار شدن خطا توسط test بعدی، زمان انجام tests، تلاش برای شناسایی خطاها، طولانی‌ترین زنجیره وابستگی در test، مدت زمان آماده‌سازی محیط test، کاهش قرار گرفتن در معرض خطر (اندازه(ضرر) × prob(ضرر))
