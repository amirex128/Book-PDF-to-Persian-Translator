جدول 13.4  راه‌حل الگوی Pipe-and-Filter

Overview
داده‌ها از ورودی‌های خارجی سیستم به خروجی‌های خارجی آن از طریق مجموعه‌ای از تبدیل‌ها که توسط filters متصل‌شده توسط pipes انجام می‌شوند، تبدیل می‌شوند.

Elements filter، که یک component است که داده‌های خوانده شده در input port(s) آن را به داده‌های نوشته شده در output port(s) آن تبدیل می‌کند. Filters می‌توانند به‌صورت همزمان با یکدیگر اجرا شوند. Filters می‌توانند داده‌ها را به‌صورت افزایشی تبدیل کنند. یعنی، آنها می‌توانند تولید خروجی را به محض شروع پردازش ورودی شروع کنند. ویژگی‌های مهم شامل نرخ‌های پردازش، فرمت‌های داده ورودی/خروجی، و تبدیل انجام شده توسط filter است.
pipe، که یک connector است که داده‌ها را از output port(s) یک filter به input port(s) یک filter دیگر منتقل می‌کند. یک pipe دارای یک منبع واحد برای ورودی خود و یک هدف واحد برای خروجی خود است. یک pipe ترتیب اقلام داده را حفظ می‌کند و داده‌های عبوری را تغییر نمی‌دهد. ویژگی‌های مهم شامل اندازه بافر، پروتکل تعامل، سرعت انتقال، و قالب داده‌هایی است که از یک pipe عبور می‌کنند.

Relations
رابطه پیوست، خروجی filters را با ورودی pipes و برعکس مرتبط می‌کند.

Constraints
Pipes، filter output ports را به filter input ports متصل می‌کنند.
Connected filters باید بر نوع داده‌ای که در طول connecting pipe منتقل می‌شود، توافق داشته باشند.
تخصیص‌های این الگو ممکن است، ارتباط components را به یک نمودار غیر چرخه‌ای یا یک توالی خطی محدود کند، که گاهی اوقات یک pipeline نامیده می‌شود.
سایر specializations ممکن است مقرر کنند که components دارای ports نام‌گذاری‌شده خاصی، مانند stdin، stdout و stderr ports از UNIX filters، باشند.

Weaknesses
الگوی pipe-and-filter معمولاً انتخاب خوبی برای یک سیستم تعاملی نیست.
داشتن تعداد زیادی filters مستقل می‌تواند مقادیر قابل توجهی از computational overhead را اضافه کند.
سیستم‌های Pipe-and-filter ممکن است برای محاسبات طولانی مدت مناسب نباشند.
