11.2  Tactics for Usability

179

یک ارتباطی بین دستیابی به usability و modifiability وجود دارد. فرآیند طراحی user interface شامل ایجاد و سپس testing یک طراحی user interface است. نقص‌ها در طراحی اصلاح می‌شوند و فرآیند تکرار می‌شود. اگر user interface قبلاً به عنوان بخشی از سیستم ساخته شده باشد، پس باید سیستم را اصلاح کرد تا آخرین طراحی منعکس شود. از این رو ارتباط با modifiability. این ارتباط منجر به الگوهای استاندارد برای پشتیبانی از طراحی user interface شده است (به بخش فرعی مراجعه کنید).

پشتیبانی از User Initiative

هنگامی که یک سیستم در حال اجرا است، usability با ارائه بازخورد به کاربر در مورد کاری که سیستم انجام می‌دهد و با اجازه دادن به کاربر برای ارائه پاسخ‌های مناسب، افزایش می‌یابد. به عنوان مثال، تاکتیک‌های توضیح داده شده در ادامه - لغو، undo، مکث/از سرگیری و تجمیع - از کاربر در تصحیح خطاها یا کارآمدتر شدن پشتیبانی می‌کنند.

Architect با برشمردن و تخصیص مسئولیت‌های سیستم برای پاسخ به دستور کاربر، یک پاسخ برای ابتکار عمل کاربر طراحی می‌کند. در اینجا چند نمونه رایج از ابتکار عمل کاربر آمده است:

  • لغو. هنگامی که کاربر یک دستور لغو را صادر می‌کند، سیستم باید به آن گوش فرا دهد (بنابراین، مسئولیت این است که یک شنونده ثابت داشته باشد که توسط اقدامات هر چیزی که لغو می‌شود، مسدود نشده باشد). دستوری که لغو می‌شود باید خاتمه یابد. هر منبعی که توسط دستور لغو شده استفاده می‌شود باید آزاد شود. و components که با دستور لغو شده همکاری می‌کنند باید مطلع شوند تا بتوانند اقدامات مناسب را نیز انجام دهند.
  • Undo. برای پشتیبانی از توانایی undo، سیستم باید مقدار کافی اطلاعات در مورد system state را حفظ کند تا بتوان state قبلی را بنا به درخواست کاربر، بازیابی کرد. چنین رکوردی ممکن است به شکل "snapshots" state - به عنوان مثال، checkpoints - یا به صورت مجموعه‌ای از عملیات برگشت‌پذیر باشد. همه عملیات را نمی‌توان به راحتی معکوس کرد: به عنوان مثال، تغییر تمام موارد حرف "a" به حرف "b" در یک سند را نمی‌توان با تغییر تمام نمونه‌های "b" به "a" معکوس کرد، زیرا ممکن است برخی از آن نمونه‌های "b" قبل از تغییر اصلی وجود داشته باشند. در این صورت سیستم باید یک رکورد مفصل‌تر از تغییر را حفظ کند. البته، برخی از عملیات‌ها، مانند زنگ زدن، را نمی‌توان undo کرد.
  • مکث/از سرگیری. هنگامی که یک کاربر یک عملیات طولانی مدت را آغاز کرده است - مثلاً، دانلود یک فایل بزرگ یا مجموعه‌ای از فایل‌ها از یک سرور - اغلب مفید است که امکان مکث و از سرگیری عملیات را فراهم کنید. مکث موثر یک عملیات طولانی مدت نیاز به توانایی آزاد کردن موقت منابع دارد تا بتوان آنها را به کارهای دیگر اختصاص داد.
