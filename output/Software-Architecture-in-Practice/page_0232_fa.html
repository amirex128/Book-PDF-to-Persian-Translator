Part Two Quality Attributes

نتیجه (و هرگونه استثنا) را به client درخواست‌کننده برمی‌گرداند. به این ترتیب، client کاملاً از هویت، مکان و ویژگی‌های server بی‌اطلاع می‌ماند. به دلیل این جدایی، اگر یک server در دسترس نباشد، یک جایگزین را می‌توان به‌طور پویا توسط broker انتخاب کرد. اگر یک server با یک service (سازگار) جایگزین شود، باز هم، broker تنها component است که باید از این تغییر مطلع شود، و بنابراین client تحت تأثیر قرار نمی‌گیرد. Proxies معمولاً علاوه بر broker به عنوان واسطه معرفی می‌شوند تا به جزئیات تعامل با broker، مانند marshaling و unmarshaling پیام‌ها، کمک کنند.

نکات منفی brokers این است که آنها پیچیدگی را اضافه می‌کنند (brokers و احتمالاً proxies باید طراحی و پیاده‌سازی شوند، همراه با پروتکل‌های پیام‌رسانی) و یک سطح از عدم indirection را بین یک client و یک server اضافه می‌کنند، که به ارتباط آنها latency اضافه می‌کند. اشکال‌زدایی brokers می‌تواند دشوار باشد زیرا آنها در محیط‌های بسیار پویا درگیر هستند که شرایط منجر به شکست ممکن است تکرار آنها دشوار باشد. Broker از دیدگاه امنیتی، نقطه حمله آشکاری خواهد بود، و بنابراین باید به‌درستی ایمن شود. همچنین یک broker، اگر با دقت طراحی نشده باشد، می‌تواند یک نقطه شکست واحد برای یک سیستم بزرگ و پیچیده باشد. و brokers به طور بالقوه می‌توانند گلوگاه‌هایی برای ارتباط باشند.

جدول 13.2 راه‌حل الگوی broker را خلاصه می‌کند.

Table 13.2  Broker Pattern Solution
Overview الگوی broker یک runtime component را تعریف می‌کند که broker نامیده می‌شود، که ارتباط بین تعدادی از clients و servers را واسطه می‌کند.
Elements Client، درخواست‌کننده services
Server، ارائه دهنده services
Broker، یک واسطه که یک server مناسب را برای برآورده کردن درخواست یک client، انتقال درخواست به server، و بازگرداندن نتایج به client، قرار می‌دهد
Client-side proxy، یک واسطه که ارتباط واقعی با broker را مدیریت می‌کند، از جمله marshaling، ارسال و unmarshaling پیام‌ها
Server-side proxy، یک واسطه که ارتباط واقعی با broker را مدیریت می‌کند، از جمله marshaling، ارسال و unmarshaling پیام‌ها
Relations رابطه پیوست، clients (و، اختیاری، client-side proxies) و servers (و، اختیاری، server-side proxies) را با brokers مرتبط می‌کند.
Constraints Client فقط می‌تواند به یک broker متصل شود (به‌طور بالقوه از طریق یک client-side proxy). Server فقط می‌تواند به یک broker متصل شود (به‌طور بالقوه از طریق یک server-side proxy).
Weaknesses Brokers یک لایه از indirection، و از این رو latency، بین clients و servers اضافه می‌کنند، و این لایه ممکن است یک گلوگاه ارتباطی باشد.

Broker ممکن است هدف حملات امنیتی باشد.
