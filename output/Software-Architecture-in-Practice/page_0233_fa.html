Part Two Quality Attributes

The broker، البته component حیاتی در این الگو است. این الگو، تمام مزایای modifiability از تاکتیک استفاده از یک واسطه (که در فصل 7 توضیح داده شد)، یک مزیت availability (زیرا الگوی broker، جایگزین کردن یک سرور از کار افتاده با یک سرور دیگر را آسان می‌کند)، و یک مزیت performance (زیرا الگوی broker، واگذاری کار به کم‌مشغول‌ترین سرور را آسان می‌کند) را ارائه می‌دهد. با این حال، این الگو نیز برخی از مسئولیت‌ها را به همراه دارد. به عنوان مثال، استفاده از یک broker، بهینه‌سازی‌های performance را که ممکن است اگر موقعیت مکانی و ویژگی‌های دقیق server را می‌دانستید، حذف می‌کند. همچنین استفاده از این الگو سربار واسطه را اضافه می‌کند و در نتیجه باعث ایجاد latency می‌شود.

نسخه اصلی الگوی broker، همان‌طور که توسط گاما، هلم، جانسون و ویلیسایدز [Gamma 94] مستند شده است، در شکل 13.6 ارائه شده است.

اولین پیاده‌سازی گسترده از الگوی broker در Common Object Request Broker Architecture (CORBA) بود. سایر موارد استفاده رایج از این الگو در Enterprise Java Beans (EJB) و پلتفرم .NET مایکروسافت یافت می‌شود - اساساً هر پلتفرم مدرن برای ارائه‌دهندگان و مصرف‌کنندگان distributed service، نوعی از broker را پیاده‌سازی می‌کند. رویکرد معماری service-oriented (SOA) به‌طور اساسی به brokers وابسته است، که معمولاً در قالب یک enterprise service bus است.
