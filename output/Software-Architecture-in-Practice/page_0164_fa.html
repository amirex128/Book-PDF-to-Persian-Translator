8—Performance

Table 8.2  Checklist to Support the Design and Analysis Process for Performance

Category Checklist
Allocation of Responsibilities مسئولیت های سیستم را که شامل بارگذاری سنگین، الزامات پاسخ زمان بحرانی، استفاده زیاد یا تأثیر بر بخش هایی از سیستم می شوند که در آن بار سنگین یا رویدادهای زمان بحرانی رخ می دهد، تعیین کنید.

برای آن مسئولیت‌ها، الزامات پردازش هر مسئولیت را شناسایی کنید و تعیین کنید که آیا ممکن است باعث ایجاد bottlenecks شوند یا خیر.

همچنین، مسئولیت های اضافی را برای شناسایی و پردازش مناسب درخواست ها شناسایی کنید، از جمله

  • مسئولیت هایی که ناشی از عبور یک thread of control از مرزهای فرآیند یا پردازنده هستند
  • مسئولیت هایی برای مدیریت threads of control— تخصیص و آزادسازی threads، حفظ thread pools و غیره
  • مسئولیت هایی برای زمانبندی منابع مشترک یا مدیریت performance-related artifacts مانند صف ها، بافرها و caches

برای مسئولیت ها و منابعی که شناسایی کردید، اطمینان حاصل کنید که پاسخ performance مورد نیاز را می توان برآورده کرد (شاید با ایجاد یک مدل performance برای کمک به ارزیابی).

Coordination Model عناصری از سیستم را که باید با یکدیگر هماهنگ شوند—مستقیم یا غیرمستقیم—تعیین کنید و مکانیسم های ارتباطی و هماهنگی را انتخاب کنید که موارد زیر را انجام دهند:
  • پشتیبانی از هر concurrency معرفی شده (به عنوان مثال، آیا thread safe است؟)، اولویت‌بندی رویداد یا استراتژی زمان‌بندی
  • اطمینان حاصل کنید که پاسخ performance مورد نیاز را می توان ارائه داد
  • می تواند ورود رویدادهای دوره ای، تصادفی یا متناوب را در صورت نیاز ثبت کند
  • دارای ویژگی های مناسب مکانیسم های ارتباطی باشد. به عنوان مثال، stateful، stateless، همزمان، ناهمزمان، تحویل تضمینی، throughput یا latency
Data Model آن بخش هایی از data model را تعیین کنید که به شدت بارگذاری می شوند، الزامات پاسخ زمان بحرانی دارند، به شدت مورد استفاده قرار می گیرند، یا بر بخش هایی از سیستم که در آن بار سنگین یا رویدادهای زمان بحرانی رخ می دهند، تأثیر می گذارند.

برای آن data abstractions، موارد زیر را تعیین کنید:

  • آیا حفظ چندین کپی از داده های کلیدی به performance کمک می کند یا خیر
  • آیا تقسیم داده ها به performance کمک می کند یا خیر
  • آیا کاهش الزامات پردازش برای ایجاد، مقداردهی اولیه، پایداری، دستکاری، ترجمه یا تخریب data abstractions های شماره گذاری شده امکان پذیر است یا خیر
  • آیا افزودن منابع برای کاهش bottlenecks برای ایجاد، مقداردهی اولیه، پایداری، دستکاری، ترجمه یا تخریب data abstractions های شماره گذاری شده امکان پذیر است یا خیر
