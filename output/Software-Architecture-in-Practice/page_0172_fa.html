9.2  Tactics for Security

Figure 9.2  The goal of security tactics

Detect Attacks

دسته detect attacks از چهار تاکتیک تشکیل شده است: detect intrusion, detect service denial, verify message integrity و detect message delay.

  • Detect intrusion مقایسه ترافیک شبکه یا الگوهای درخواست سرویس در داخل یک سیستم با مجموعه ای از signatures یا الگوهای شناخته شده رفتار مخرب است که در یک database ذخیره شده اند. The signatures می توانند بر اساس protocol, TCP flags, payload sizes, applications, source یا destination address، یا port number باشند.
  • Detect service denial مقایسه الگو یا signature ترافیک شبکه وارد شده به یک سیستم با profileهای تاریخی حملات denial-of-service شناخته شده است.
  • Verify message integrity. این تاکتیک از تکنیک هایی مانند checksums یا hash values برای تأیید integrity پیام ها، resource filesها، deployment filesها و configuration filesها استفاده می کند. A checksum یک مکانیسم اعتبارسنجی است که در آن سیستم اطلاعات اضافی را برای configuration files و پیام ها نگه می دارد و از این اطلاعات اضافی برای تأیید configuration file یا message هنگام استفاده از آن استفاده می کند. A hash value یک رشته منحصر به فرد است که توسط یک hashing function ایجاد می شود که ورودی آن می تواند configuration files یا پیام ها باشد. حتی یک تغییر جزئی در files یا پیام های اصلی منجر به تغییر قابل توجهی در hash value می شود.
  • Detect message delay برای تشخیص حملات potential man-in-the-middle در نظر گرفته شده است، جایی که یک party مخرب در حال رهگیری (و احتمالاً اصلاح) پیام ها است. با بررسی زمانی که برای تحویل یک پیام طول می کشد، می توان رفتار timing مشکوک را تشخیص داد، جایی که زمان تحویل یک پیام بسیار متغیر است.
