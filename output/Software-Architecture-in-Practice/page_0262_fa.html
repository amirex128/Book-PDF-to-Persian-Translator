  • Performance. سطوح indirection بین client (درخواست اطلاعات یا service) و server (ارائه اطلاعات یا service) overhead را اضافه می‌کند، و از این رو latency را افزایش می‌دهد. همچنین، broker یک performance bottleneck بالقوه است، اگر ارتباط مستقیم بین client و server مطلوب نباشد (به عنوان مثال، به دلایل امنیتی).
  • Testability. Brokers در سیستم‌های پیچیده چند فرآیندی و چند پردازنده‌ای استفاده می‌شوند. چنین سیستم‌هایی معمولاً بسیار پویا هستند. درخواست‌ها و پاسخ‌ها معمولاً ناهمزمان هستند. همه اینها، testing و debugging این سیستم‌ها را بسیار دشوار می‌کند. اما شرح الگوی broker هیچ عملکرد testing، مانند testing interfaces، قابلیت‌های ثبت و پخش state یا فعالیت و غیره را ارائه نمی‌دهد.
