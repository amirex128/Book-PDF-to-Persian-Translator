Part Two Quality Attributes

هنگامی که شما یک مدل برای QA خود دارید، می‌توانید برای فهرست کردن architectural approaches (تاکتیک‌ها و الگوها) که برای دستکاری هر یک از پارامترهای مربوطه به نفع خودتان باز هستند، کار کنید.

Assemble a Set of Tactics for the New Quality Attribute

دو منبع وجود دارد که می‌توان از آنها برای استخراج تاکتیک‌ها برای هر quality attribute استفاده کرد: مدل‌ها و متخصصان.

شکل 12.2 یک مدل صف برای performance را نشان می‌دهد. این مدل‌ها به‌طور گسترده‌ای برای تجزیه و تحلیل latency و throughput انواع مختلف سیستم‌های صف، از جمله محیط‌های تولید و خدمات، و همچنین سیستم‌های کامپیوتری استفاده می‌شوند.

در این مدل، هفت پارامتر وجود دارد که می‌توانند بر latency که مدل پیش‌بینی می‌کند، تأثیر بگذارند:

  • نرخ ورود
  • Queuing discipline
  • الگوریتم زمان‌بندی
  • زمان Service
  • Topology
  • پهنای باند شبکه
  • الگوریتم مسیریابی
