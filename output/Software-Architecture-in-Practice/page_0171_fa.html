9 — Security

جدول 9.1 سناریوی کلی امنیت

بخش سناریو مقادیر ممکن منبع
Human یا یک سیستم دیگر که ممکن است قبلاً شناسایی شده باشد (چه درست و چه نادرست) یا ممکن است در حال حاضر ناشناخته باشد. یک human attacker ممکن است از خارج از سازمان یا از داخل سازمان باشد.
Stimulus تلاش غیرمجاز برای نمایش داده‌ها، تغییر یا حذف داده‌ها، دسترسی به system services، تغییر رفتار سیستم یا کاهش availability.
Artifact System services, داده‌ها در داخل سیستم، یک component یا منابع سیستم، داده‌های تولید شده یا مصرف شده توسط سیستم
Environment سیستم یا آنلاین است یا آفلاین؛ یا به یک شبکه متصل است یا از آن جدا شده است؛ یا پشت یک firewall است یا به یک شبکه باز است؛ کاملاً عملیاتی، تا حدی عملیاتی یا غیر عملیاتی.
Response تراکنش‌ها به گونه‌ای انجام می‌شوند که:
  • داده‌ها یا services از دسترسی غیرمجاز محافظت می‌شوند.
  • داده‌ها یا services بدون مجوز دستکاری نمی‌شوند.
  • طرفین یک تراکنش با اطمینان شناسایی می‌شوند.
  • طرفین معامله نمی‌توانند از مشارکت خود سرباز زنند.
  • داده‌ها، منابع و system services برای استفاده قانونی در دسترس خواهند بود.
سیستم فعالیت‌های درون خود را پیگیری می‌کند با:
  • ثبت دسترسی یا اصلاح
  • ثبت تلاش‌ها برای دسترسی به داده‌ها، منابع یا services
  • اعلام به اشخاص (افراد یا سیستم‌ها) مناسب در صورت وقوع یک حمله آشکار
Response Measure یک یا چند مورد از موارد زیر:
  • چقدر از سیستم در صورت به خطر افتادن یک component یا مقدار داده خاص به خطر می‌افتد
  • چقدر زمان سپری شد قبل از اینکه یک حمله شناسایی شود
  • به چند حمله مقاومت شد
  • چقدر طول می‌کشد تا از یک حمله موفقیت‌آمیز بازیابی شود
  • چه مقدار از داده‌ها در برابر یک حمله خاص آسیب‌پذیر است

9.2 Tactics for Security

یک روش برای فکر کردن در مورد چگونگی دستیابی به security در یک سیستم، فکر کردن در مورد physical security است. نصب‌های امن دسترسی محدودی دارند (به عنوان مثال، با استفاده از security checkpoints)، ابزارهایی برای تشخیص مزاحمان دارند (به عنوان مثال، با درخواست از بازدیدکنندگان قانونی برای استفاده از badges)، مکانیسم‌های بازدارنده مانند نگهبانان مسلح دارند، مکانیسم‌های واکنشی مانند قفل شدن خودکار درها دارند، و مکانیسم‌های بازیابی مانند پشتیبان‌گیری خارج از سایت دارند. اینها منجر به چهار دسته تاکتیک ما می‌شوند: detect, resist, react, و recover. شکل 9.2 این دسته‌ها را به عنوان هدف تاکتیک‌های security نشان می‌دهد.
