  • مرورگرهای وب تا زمانی که درخواست داده سرویس‌دهی نشده است، مسدود نمی‌شوند.
  • در برخی از الگوهای client-server، به servers اجازه داده می‌شود تا اقدامات خاصی را در clients خود آغاز کنند. این کار را می‌توان با اجازه دادن به یک client برای ثبت روش‌های notification، یا callbacks، که server در زمان‌های خاصی فراخوانی می‌کند، انجام داد.
  • در سیستم‌های دیگر، service calls از طریق یک request/reply connector با یک "جلسه" که شروع و پایان مجموعه‌ای از تعامل client-server را مشخص می‌کند، محصور می‌شوند.

الگوی client-server، برنامه‌های client را از services که استفاده می‌کنند جدا می‌کند. این الگو با فاکتور کردن services مشترک، که قابل استفاده مجدد هستند، سیستم‌ها را ساده می‌کند. از آنجایی که servers می‌توانند توسط هر تعدادی از clients، دسترسی داشته باشند، افزودن clients جدید به یک سیستم آسان است. به طور مشابه، servers را می‌توان برای پشتیبانی از scalability یا availability تکرار کرد.

World Wide Web، شناخته‌شده‌ترین نمونه از سیستمی است که بر اساس الگوی client-server است، که به clients (مرورگرهای وب) اجازه می‌دهد تا با استفاده از HyperText Transfer Protocol (HTTP) به اطلاعات از servers در سراسر اینترنت دسترسی داشته باشند. HTTP یک پروتکل request/reply است. HTTP بدون state است. اتصال بین client و server پس از هر پاسخ از server خاتمه می‌یابد.
