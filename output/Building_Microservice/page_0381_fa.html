User credentials (اعتبارنامه‌های کاربر)

User credentials (اعتبارنامه‌های کاربر)، مانند ترکیبی از ایمیل و رمز عبور، برای نحوه کار بسیاری از ما با نرم‌افزارمان ضروری باقی می‌مانند، اما آنها همچنین یک نقطه ضعف بالقوه هستند که به دسترسی سیستم‌های ما توسط افراد مخرب مربوط می‌شود. گزارش تحقیقات نقض داده‌های ورایزون در سال 2020 نشان داد که نوعی از سرقت credential (اعتبارنامه) در 80٪ از موارد ناشی از هک استفاده شده است. این شامل موقعیت‌هایی است که در آنها credentials (اعتبارنامه‌ها) از طریق مکانیسم‌هایی مانند حملات فیشینگ یا جایی که رمزهای عبور به صورت brute-forced (زورگیرانه) به دست آمده‌اند، به سرقت رفته‌اند.

توصیه‌های عالی در مورد نحوه رسیدگی صحیح به مواردی مانند رمزهای عبور وجود دارد—توصیه‌هایی که با وجود سادگی و وضوح در پیروی، هنوز به اندازه کافی به‌طور گسترده اتخاذ نمی‌شوند. Troy Hunt یک نمای کلی عالی از آخرین توصیه‌ها از هر دو NIST و مرکز ملی Cyber Security بریتانیا دارد.6 این توصیه‌ها شامل توصیه‌هایی برای استفاده از password managers (مدیریت‌کننده‌های رمز عبور) و رمزهای عبور طولانی، اجتناب از استفاده از قوانین پیچیده رمز عبور، و—تا حدودی تعجب‌آور—اجتناب از تغییرات منظم رمز عبور است. پست کامل Troy ارزش خواندن دقیق را دارد.

در عصر کنونی سیستم‌های مبتنی بر API، credentials (اعتبارنامه‌های) ما همچنین به مدیریت مواردی مانند کلیدهای API برای سیستم‌های شخص ثالث، مانند حساب‌هایی برای ارائه دهنده cloud (ابری) شما، گسترش می‌یابد. به عنوان مثال، اگر یک فرد مخرب به حساب ریشه AWS شما دسترسی پیدا کند، می‌تواند تصمیم بگیرد همه چیز را که در آن حساب در حال اجرا است، از بین ببرد. در یک نمونه افراطی، چنین حمله‌ای منجر به ورشکستگی شرکتی به نام Code Spaces شد7—تمام منابع آنها در یک حساب واحد اجرا می‌شدند، همه‌چیز از جمله backups (پشتیبان‌گیری). طعنه Code Spaces که "Rock Solid, Secure and Affordable Svn Hosting, Git Hosting and Project Management" را ارائه می‌داد، از نظر من پنهان نیست.

حتی اگر کسی کلیدهای API شما را برای ارائه‌دهنده cloud (ابری) شما به دست آورد و تصمیم به از بین بردن همه چیزهایی که ساخته‌اید نگیرد، ممکن است تصمیم بگیرد که برخی از ماشین‌های مجازی گران‌قیمت را راه‌اندازی کند تا مقداری bitcoin mining (استخراج بیت‌کوین) را اجرا کند به این امید که شما متوجه نشوید. این اتفاق برای یکی از مشتریان من افتاد، که متوجه شد کسی قبل از خاموش شدن حساب، بیش از 10 هزار دلار برای انجام این کار هزینه کرده است. معلوم شد که مهاجمان نیز می‌دانند چگونه automation (خودکارسازی) کنند—bots (ربات‌هایی) وجود دارند که فقط credentials (اعتبارنامه‌ها) را اسکن می‌کنند و سعی می‌کنند از آنها برای راه‌اندازی ماشین‌ها برای استخراج ارزهای رمزنگاری شده استفاده کنند.
