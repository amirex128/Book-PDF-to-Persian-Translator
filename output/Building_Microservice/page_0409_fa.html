Format (قالب)

payload (بار) اصلی یک JWT، یک ساختار JSON است، که به طور کلی می‌تواند شامل هر چیزی باشد که شما می‌خواهید. ما می‌توانیم یک نمونه از توکن را در مثال 11-1 مشاهده کنیم. استاندارد JWT (توکن وب JSON) برخی از فیلدهای خاص نامگذاری شده ("public claims" (ادعاهای عمومی)) را توصیف می‌کند که شما باید در صورت مرتبط بودن از آنها استفاده کنید. به عنوان مثال، exp تاریخ انقضای یک توکن را تعریف می‌کند. اگر شما از این فیلدهای public claim (ادعای عمومی) به درستی استفاده کنید، احتمال زیادی وجود دارد که کتابخانه‌هایی که استفاده می‌کنید بتوانند به درستی از آنها استفاده کنند—به عنوان مثال، رد کردن یک توکن اگر فیلد exp (انقضا) نشان می‌دهد که توکن قبلاً منقضی شده است.16 حتی اگر شما از همه این public claims (ادعاهای عمومی) استفاده نکنید، آگاهی از اینکه آنها چه هستند ارزش دارد تا اطمینان حاصل کنید که در نهایت از آنها برای استفاده‌های خاص برنامه خود استفاده نمی‌کنید، زیرا این می‌تواند باعث ایجاد رفتارهای عجیبی در کتابخانه‌های پشتیبانی شود.

مثال 11-1. نمونه‌ای از payload (بار) JSON یک JWT

{ "sub": "123", "name": "Sam Newman", "exp": 1606741736, "groups": "admin, beta" }
