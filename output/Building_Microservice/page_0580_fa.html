سبک‌های ارتباطی (ادامه)

شکل A-2. یک microservice که عملکرد خود را از طریق یک REST API و یک موضوع ارائه می‌دهد

همکاری مبتنی بر رویداد می‌تواند ایجاد معماری‌های loosely coupledتر را آسان‌تر کند، اما می‌تواند به کار بیشتری برای درک نحوه عملکرد سیستم نیاز داشته باشد. این نوع ارتباط اغلب مستلزم استفاده از فناوری‌های تخصصی مانند message brokers است، که می‌تواند مسائل را پیچیده‌تر کند. اگر شما می‌توانید از یک message broker کاملاً مدیریت شده استفاده کنید، که این امر می‌تواند به کاهش هزینه این نوع سیستم‌ها کمک کند.

مدل‌های تعاملی request-response و مبتنی بر رویداد هر دو جایگاه خود را دارند، و اغلب اینکه شما از کدام یک استفاده می‌کنید، یک ترجیح شخصی خواهد بود. برخی از مشکلات فقط به یک مدل بیشتر از دیگری می‌پردازند، و برای یک معماری microservice رایج است که ترکیبی از سبک‌ها را داشته باشد.
