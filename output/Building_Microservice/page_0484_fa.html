شکل 14-2. UI از هم جدا شده و متعلق به تیمی است که عملکرد سمت سرور را که از UI پشتیبانی می‌کند نیز مدیریت می‌کند

عوامل محرک برای تیم‌های Frontend اختصاصی

به نظر می‌رسد که تمایل به تیم‌های frontend اختصاصی به سه عامل اصلی برمی‌گردد: کمبود متخصصان، تمایل به ثبات و چالش‌های فنی.

اولاً، ارائه یک رابط کاربری نیازمند درجه‌ای از مهارت‌های تخصصی است. جنبه‌های تعامل و طراحی گرافیکی وجود دارد، و سپس دانش فنی لازم برای ارائه یک تجربه وب یا برنامه بومی عالی وجود دارد. متخصصانی با این مهارت‌ها می‌توانند به سختی به دست آیند، و از آنجایی که این افراد یک کالای کمیاب هستند، وسوسه می‌شود که همه آنها را در کنار هم قرار دهید تا بتوانید مطمئن شوید که فقط روی تخصص خود تمرکز می‌کنند.

دومین عامل محرک برای یک تیم frontend جداگانه، ثبات است. اگر شما یک تیم مسئول ارائه رابط کاربری مشتری‌محور خود دارید، می‌توانید اطمینان حاصل کنید که UI شما ظاهری ثابت دارد. شما از مجموعه ثابتی از کنترل‌ها برای حل مشکلات مشابه استفاده می‌کنید تا رابط کاربری مانند یک موجودیت واحد و منسجم به نظر برسد.
