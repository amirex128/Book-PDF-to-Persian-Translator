What you’ll learn in this book

با انتخاب‌های فناوری که در ابتدای پروژه انجام داده‌اند، گیر کرده‌اند. اغلب، آن‌ها باید برنامه‌ای را که با استفاده از یک technology stack به طور فزاینده‌ای منسوخ شده است، نگهداری کنند.

فریم‌ورک Spring همچنان به تکامل خود ادامه می‌دهد در حالی که با عقب سازگار است، بنابراین از نظر تئوری FTGO ممکن است قادر به ارتقا بوده باشد. متأسفانه، application FTGO از نسخه‌های فریم‌ورک‌هایی استفاده می‌کند که با نسخه‌های جدیدتر Spring ناسازگار هستند. تیم توسعه هرگز زمان لازم برای ارتقای آن فریم‌ورک‌ها را پیدا نکرده است. در نتیجه، بخش‌های اصلی application با استفاده از فریم‌ورک‌های منسوخ نوشته شده‌اند. علاوه بر این، developers FTGO می‌خواهند زبان‌های غیر JVM مانند GoLang و NodeJS را آزمایش کنند. متأسفانه، این کار با یک application monolithic امکان‌پذیر نیست.

1.2

Why this book is relevant to you

احتمالاً شما یک developer، architect، CTO یا VP مهندسی هستید. شما مسئول یک application هستید که از معماری monolithic خود فراتر رفته است. مانند ماری در FTGO، شما با تحویل software دست و پنجه نرم می‌کنید و می‌خواهید بدانید چگونه از جهنم monolith فرار کنید. یا شاید شما می‌ترسید که سازمان شما در مسیر جهنم monolithic قرار دارد و می‌خواهید بدانید چگونه جهت را تغییر دهید قبل از اینکه دیر شود. اگر نیاز دارید از جهنم monolithic فرار کنید یا از آن اجتناب کنید، این کتاب برای شماست.

این کتاب زمان زیادی را صرف توضیح مفاهیم معماری microservice می‌کند. هدف من این است که شما این مطالب را بدون توجه به technology stackی که استفاده می‌کنید، در دسترس بدانید. تنها چیزی که نیاز دارید این است که با اصول اولیه معماری و طراحی application سازمانی آشنا باشید. به طور خاص، شما باید موارد زیر را بدانید:

  • معماری Three-tier
  • طراحی Web application
  • نحوه توسعه business logic با استفاده از طراحی شی گرا
  • نحوه استفاده از RDBMS: SQL و ACID transactions
  • نحوه استفاده از interprocess communication با استفاده از یک message broker و REST APIs
  • امنیت، از جمله authentication و authorization

مثال‌های کد در این کتاب با استفاده از Java و فریم‌ورک Spring نوشته شده‌اند. این بدان معناست که برای اینکه بیشترین بهره را از مثال‌ها ببرید، باید با فریم‌ورک Spring نیز آشنا باشید.
