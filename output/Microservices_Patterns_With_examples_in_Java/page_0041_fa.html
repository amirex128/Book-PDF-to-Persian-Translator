Microservice architecture to the rescue

یک service یک mini application است که عملکردی با تمرکز محدود را پیاده‌سازی می‌کند، مانند مدیریت سفارش، مدیریت مشتری و غیره. یک service با استفاده از مقیاس‌بندی محور X مقیاس‌بندی می‌شود، اگرچه برخی از services ممکن است از مقیاس‌بندی محور Z نیز استفاده کنند. به عنوان مثال، Order service شامل مجموعه‌ای از نمونه‌های service با load-balanced است.

تعریف سطح بالای معماری microservice (microservices) یک سبک معماری است که یک application را به طور عملکردی به مجموعه‌ای از services تجزیه می‌کند. توجه داشته باشید که این تعریف چیزی در مورد اندازه نمی‌گوید. در عوض، چیزی که مهم است این است که هر service مجموعه‌ای متمرکز و منسجم از مسئولیت‌ها را دارد. در ادامه کتاب در مورد معنای آن بحث خواهم کرد.

اکنون بیایید نگاهی بیندازیم که چگونه معماری microservice یک نوع ماژولار است.

1.4.2

Microservices as a form of modularity

ماژولار بودن در هنگام توسعه applications بزرگ و پیچیده ضروری است. یک application مدرن مانند FTGO برای توسعه توسط یک فرد بسیار بزرگ است. همچنین برای اینکه توسط یک فرد واحد درک شود، بسیار پیچیده است. Applications باید به ماژول‌هایی تجزیه شوند که توسط افراد مختلف توسعه یافته و درک می‌شوند. در یک application monolithic، ماژول‌ها با استفاده از ترکیبی از سازه‌های زبان برنامه‌نویسی (مانند بسته‌های Java) و مصنوعات ساخت (مانند فایل‌های Java JAR) تعریف می‌شوند. با این حال، همانطور که developers FTGO کشف کرده‌اند، این رویکرد در عمل خوب عمل نمی‌کند. applications monolithic طولانی‌مدت معمولاً به توپ‌های بزرگ گل تبدیل می‌شوند.
