Interprocess communication در یک معماری microservice

این رویکرد به service discovery ترکیبی از دو pattern است. اولین pat- tern، الگوی Self registration است. یک نمونه سرویس، API ثبت service registry را فراخوانی می‌کند برای ثبت موقعیت شبکه خود. همچنین ممکن است یک health check ارائه دهد URL، که در فصل 11 با جزئیات بیشتری توضیح داده شده است. URL health check یک API end- point است که service registry به صورت دوره‌ای برای تأیید اینکه نمونه سرویس، سالم است و برای رسیدگی به requests در دسترس است. یک service registry ممکن است به یک سرویس نیاز داشته باشد نمونه برای فراخوانی دوره‌ای یک API “heartbeat” به منظور جلوگیری از ثبت نام آن منقضی شدن.

الگوی دوم، الگوی Client-side discovery است. هنگامی که یک کلاینت سرویس می‌خواهد یک سرویس را فراخوانی کند، service registry را query می‌کند تا لیستی از نمونه‌های سرویس را دریافت کند. برای بهبود عملکرد، یک کلاینت ممکن است نمونه‌های سرویس را cache کند. کلاینت سرویس
