Using the CQRS pattern

در یک service مبتنی بر CQRS، مدل domain سمت دستور عملیات CRUD را مدیریت می‌کند و به database خود نگاشت می‌شود. همچنین ممکن است queries ساده، مانند queries غیر join، مبتنی بر primary key را مدیریت کند. سمت دستور، هر زمان که داده‌هایش تغییر می‌کند، domain events را منتشر می‌کند. این events ممکن است با استفاده از یک فریم‌ورک مانند Eventuate Tram یا با استفاده از event sourcing منتشر شوند.

یک مدل query جداگانه، queries غیر بدیهی را مدیریت می‌کند. این مدل بسیار ساده‌تر از سمت دستور است زیرا مسئول پیاده‌سازی قوانین تجاری نیست. سمت query از هر نوع databaseای که برای queriesی که باید پشتیبانی کند، منطقی باشد، استفاده می‌کند. سمت query دارای event handlers است که به domain events مشترک می‌شوند و database یا databases را به‌روزرسانی می‌کنند. حتی ممکن است چندین مدل query وجود داشته باشد، یکی برای هر نوع query.
