فصل 11

Developing production-ready services

Metrics به طور دوره‌ای sample می‌شوند. A metric sample دارای سه property زیر است:

Name—The name از metric، مانند jvm_memory_max_bytes یا placed_orders

Value—A numeric value

Timestamp—The time از sample

In addition، برخی از monitoring systems از مفهوم dimensions پشتیبانی می‌کنند، که arbitrary name-value pairs هستند. For example, jvm_memory_max_bytes با dimen- sions مانند area="heap",id="PS Eden Space" و area="heap",id="PS Old Gen" گزارش می‌شود.

Dimensions اغلب برای ارائه اطلاعات اضافی، مانند machine استفاده می‌شود name یا service name، یا یک service instance identifier. A monitoring system معمولاً metric samples را در امتداد یک یا چند dimension جمع‌آوری (جمع یا میانگین‌گیری) می‌کند.

Many aspects از monitoring the responsibility از operations است. But a service developer مسئول دو جنبه از metrics است. First, they must instrument their service تا metrics را در مورد behavior خود جمع‌آوری کند. Second، آنها باید those را expose کنند service metrics، به همراه metrics از JVM و the application framework، to the metrics server.
