DATA CONSISTENCY PATTERNS FOR IMPLEMENTING TRANSACTION MANAGEMENT

همانطور که قبلاً ذکر شد، برای اطمینان از loose coupling، هر service پایگاه داده خود را دارد. متأسفانه، داشتن یک پایگاه داده در هر service، برخی از مسائل مهم را معرفی می‌کند. من در فصل 4 شرح می‌دهم که رویکرد سنتی استفاده از transactions توزیع شده (2PC) یک گزینه مناسب برای یک application مدرن نیست. در عوض، یک application باید با استفاده از الگوی Saga، سازگاری داده‌ها را حفظ کند. شکل 1.13 الگوهای مرتبط با داده را نشان می‌دهد.
