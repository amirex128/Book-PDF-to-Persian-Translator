The design of the Order Service (طراحی Order Service) و the Create Order Saga (حماسه ایجاد سفارش)

  1. The SagaManager (مدیر حماسه) command message (پیام دستور) را به participant (شرکت‌کننده) saga (حماسه) (Consumer Service (سرویس مصرف‌کننده)) ارسال می‌کند.
  2. The SagaManager (مدیر حماسه) نمونه saga (حماسه) را در database (پایگاه داده) ذخیره می‌کند.

شکل 4.14 توالی events (رویدادها) را زمانی نشان می‌دهد که SagaManager (مدیر حماسه) یک reply (پاسخ) را از Consumer Service (سرویس مصرف‌کننده) دریافت می‌کند.

توالی events (رویدادها) به شرح زیر است:

  1. Eventuate Tram (ترام Eventuate) SagaManager (مدیر حماسه) را با reply (پاسخ) از Consumer Service (سرویس مصرف‌کننده) فراخوانی می‌کند.
  2. SagaManager (مدیر حماسه) نمونه saga (حماسه) را از database (پایگاه داده) بازیابی می‌کند.
  3. SagaManager (مدیر حماسه) گام بعدی تعریف saga (حماسه) را اجرا می‌کند.
