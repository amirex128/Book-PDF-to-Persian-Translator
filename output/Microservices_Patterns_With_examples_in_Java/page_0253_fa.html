Querying using the API composition pattern

عملیات query با فراخوانی servicesی که مالک data هستند و ترکیب نتایج. شکل 7.2 ساختار این الگو را نشان می‌دهد. این الگو دارای دو نوع شرکت‌کننده است:

  • یک API composer— این عملیات query را با querying provider services پیاده‌سازی می‌کند.
  • یک provider service— این یک service است که مالک بخشی از data است که query برمی‌گرداند.

شکل 7.2 سه provider services را نشان می‌دهد. API composer، query را با بازیابی data از provider services و ترکیب نتایج، پیاده‌سازی می‌کند. یک API composer ممکن است یک client باشد، مانند یک application وب، که برای رندر کردن یک صفحه وب به data نیاز دارد. یا، ممکن است یک service باشد، مانند یک API gateway و Backends for frontends آن که در فصل 8 توضیح داده شد، که عملیات query را به عنوان یک API endpoint در معرض دید قرار می‌دهد.

اینکه آیا می‌توانید از این الگو برای پیاده‌سازی یک عملیات query خاص استفاده کنید، به عوامل متعددی بستگی دارد، از جمله نحوه تقسیم data، قابلیت‌های APIs ارائه شده توسط servicesی که مالک data هستند، و قابلیت‌های databasesی که توسط services استفاده می‌شوند. به عنوان مثال، حتی اگر Provider services دارای APIs برای بازیابی
