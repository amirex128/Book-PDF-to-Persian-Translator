درباره این کتاب

هدف از این کتاب این است که به شما آموزش دهد چگونه با موفقیت applications را با استفاده از معماری Microservice توسعه دهید.

این کتاب نه تنها در مورد مزایای معماری Microservice بحث می‌کند، بلکه معایب آن را نیز توضیح می‌دهد. شما یاد خواهید گرفت که چه زمانی باید استفاده از معماری Monolithic را در نظر بگیرید و چه زمانی استفاده از Microservices منطقی است.

چه کسانی باید این کتاب را بخوانند

تمرکز این کتاب بر معماری و توسعه است. این کتاب برای هر کسی که مسئول توسعه و ارائه software است، مانند developers، architects، CTOs یا VPs مهندسی در نظر گرفته شده است.

این کتاب بر توضیح Microservice architecture patterns و سایر مفاهیم متمرکز است. هدف من این است که شما این مطالب را بدون توجه به technology stackی که استفاده می‌کنید، در دسترس بیابید. شما فقط باید با اصول اولیه معماری و طراحی enterprise application آشنا باشید. به طور خاص، شما باید مفاهیمی مانند معماری three-tier، طراحی web application، پایگاه‌های داده رابطه‌ای، interprocess communication با استفاده از messaging و REST و اصول اولیه امنیت application را درک کنید. با این حال، مثال‌های کد از Java و فریم‌ورک Spring استفاده می‌کنند. برای اینکه بیشترین بهره را از آن‌ها ببرید، باید با فریم‌ورک Spring آشنا باشید.
