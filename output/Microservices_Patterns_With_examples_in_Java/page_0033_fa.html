1.1.1

The architecture of the FTGO application

FTGO یک application Java سازمانی معمولی است. شکل 1.1 معماری آن را نشان می‌دهد. application FTGO دارای یک معماری شش ضلعی است، که یک سبک معماری است که در فصل 2 با جزئیات بیشتری توضیح داده شده است. در یک معماری شش ضلعی، هسته application از business logic تشکیل شده است. اطراف business logic آداپتورهای مختلفی وجود دارد که رابط‌های کاربری را پیاده‌سازی می‌کنند و با سیستم‌های خارجی ادغام می‌شوند.

business logic شامل ماژول‌هایی است که هر کدام مجموعه‌ای از domain objects هستند. نمونه‌هایی از این ماژول‌ها شامل Order Management، Delivery Management، Billing و Payments است. چندین آداپتور وجود دارد که با سیستم‌های خارجی ارتباط برقرار می‌کنند. برخی آداپتورهای inbound هستند که درخواست‌ها را با فراخوانی business logic، از جمله REST API و آداپتورهای Web UI، مدیریت می‌کنند. برخی دیگر آداپتورهای outbound هستند، که business logic را قادر می‌سازند تا به پایگاه داده MySQL دسترسی پیدا کرده و سرویس‌های ابری مانند Twilio و Stripe را فراخوانی کنند.

علیرغم داشتن یک معماری منطقی ماژولار، application FTGO به عنوان یک فایل WAR واحد بسته‌بندی می‌شود. این application نمونه‌ای از سبک monolithic است که به طور گسترده استفاده می‌شود.

شکل 1.1

application FTGO دارای معماری شش ضلعی است. این معماری از business logic تشکیل شده است که توسط آداپتورهایی احاطه شده است که رابط‌های کاربری را پیاده‌سازی می‌کنند و با سیستم‌های خارجی، مانند mobile applications و سرویس‌های ابری برای پرداخت‌ها، پیام‌رسانی و ایمیل، ارتباط برقرار می‌کنند.
