پیشگفتار

هدف از این کتاب پاسخ به این سوال است. این کتاب درک خوبی از معماری Microservice، مزایا و معایب آن و زمان استفاده از آن به شما ارائه می‌دهد.

این کتاب نحوه حل چالش‌های طراحی متعددی را که با آن‌ها روبرو خواهید شد، از جمله نحوه مدیریت داده‌های توزیع‌شده، شرح می‌دهد. همچنین نحوه refactor کردن یک application Monolith به معماری Microservice را پوشش می‌دهد. اما این کتاب یک manifesto Microservices نیست.

در عوض، حول مجموعه‌ای از patterns سازماندهی شده است. یک pattern یک راه‌حل قابل استفاده مجدد برای یک مشکل است که در یک زمینه خاص رخ می‌دهد. زیبایی یک pattern این است که علاوه بر توصیف مزایای راه‌حل، معایب و مشکلاتی را که باید برای پیاده‌سازی موفقیت‌آمیز یک راه‌حل برطرف کنید، نیز شرح می‌دهد. بر اساس تجربه‌ام، این نوع عینیت در تفکر در مورد راه‌حل‌ها منجر به تصمیم‌گیری بسیار بهتر می‌شود. امیدوارم از خواندن این کتاب لذت ببرید و به شما بیاموزد که چگونه Microservices را با موفقیت توسعه دهید.
