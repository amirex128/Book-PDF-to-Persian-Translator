Microservice architecture to the rescue

این کتاب یک مدل مقیاس‌پذیری سه‌بعدی مفید را توصیف می‌کند: scale cube، که در شکل 1.3 نشان داده شده است.

این مدل سه روش برای مقیاس‌بندی یک application را تعریف می‌کند: X، Y و Z.

X-AXIS SCALING LOAD BALANCES REQUESTS ACROSS MULTIPLE INSTANCES

مقیاس‌بندی محور X یک راه رایج برای مقیاس‌بندی یک application monolithic است. شکل 1.4 نحوه عملکرد مقیاس‌بندی محور X را نشان می‌دهد. شما چندین نمونه از application را در پشت یک load balancer اجرا می‌کنید. load balancer درخواست‌ها را بین N نمونه یکسان از application توزیع می‌کند. این یک راه عالی برای بهبود ظرفیت و در دسترس بودن یک application است.

Z-AXIS SCALING ROUTES REQUESTS BASED ON AN ATTRIBUTE OF THE REQUEST

مقیاس‌بندی محور Z همچنین چندین نمونه از application monolith را اجرا می‌کند، اما بر خلاف مقیاس‌بندی محور X، هر نمونه فقط مسئول زیرمجموعه‌ای از داده‌ها است. شکل 1.5 نحوه عملکرد مقیاس‌بندی محور Z را نشان می‌دهد. مسیریاب جلوی نمونه‌ها از یک ویژگی درخواست برای هدایت آن به نمونه مناسب استفاده می‌کند. به عنوان مثال، یک application ممکن است درخواست‌ها را با استفاده از userId مسیریابی کند.

در این مثال، هر نمونه application مسئول زیرمجموعه‌ای از کاربران است. مسیریاب از userId مشخص شده توسط سربرگ Authorization درخواست برای انتخاب یکی از
