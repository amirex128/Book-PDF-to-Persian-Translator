درباره این کتاب

Roadmap

این کتاب شامل 13 فصل است:

  • فصل 1 علائم جهنم Monolithic را توصیف می‌کند، که زمانی رخ می‌دهد که یک application از معماری خود فراتر می‌رود، و در مورد چگونگی فرار با اتخاذ معماری Microservice توصیه می‌کند. همچنین مروری بر زبان Microservice architecture pattern ارائه می‌دهد، که موضوع سازمان‌دهی برای اکثر کتاب است.
  • فصل 2 توضیح می‌دهد که چرا معماری software مهم است و patternsی را که می‌توانید برای تجزیه یک application به مجموعه‌ای از services استفاده کنید، شرح می‌دهد. همچنین توضیح می‌دهد که چگونه بر موانع مختلفی که معمولاً در این مسیر با آن مواجه می‌شوید غلبه کنید.
  • فصل 3 الگوهای مختلف برای ارتباط محکم و بین فرآیندی را در معماری Microservice توصیف می‌کند. توضیح می‌دهد که چرا ارتباط ناهمزمان و مبتنی بر پیام اغلب بهترین انتخاب است.
  • فصل 4 توضیح می‌دهد که چگونه با استفاده از Saga pattern، سازگاری داده‌ها را در services حفظ کنید. یک saga دنباله‌ای از transactions محلی است که با استفاده از messaging ناهمزمان هماهنگ شده‌اند.
  • فصل 5 نحوه طراحی business logic را برای یک service با استفاده از الگوهای رویداد Domain-driven design (DDD) Aggregate و Domain توضیح می‌دهد.
  • فصل 6 بر اساس فصل 5 ساخته شده است و نحوه توسعه business logic را با استفاده از Event sourcing pattern توضیح می‌دهد، یک راه ساختار یافته event-centric برای ساختار business logic و پایداری domain objects.
  • فصل 7 نحوه پیاده‌سازی queries را توصیف می‌کند که داده‌های پراکنده در چندین service را با استفاده از API composition pattern یا الگوی Command query responsibility segregation (CQRS) بازیابی می‌کند.
  • فصل 8 API patterns خارجی را برای رسیدگی به درخواست‌ها از مجموعه‌ای متنوع از clients خارجی، مانند applications تلفن همراه، applications مبتنی بر browser-based JavaScript و applications شخص ثالث پوشش می‌دهد.
  • فصل 9 اولین فصل از دو فصل در مورد تکنیک‌های تست خودکار برای Microservices است. مفاهیم مهم تست مانند هرم تست را معرفی می‌کند، که نسبت نسبی هر نوع تست را در test suite شما توصیف می‌کند. همچنین نشان می‌دهد که چگونه unit tests را بنویسید، که پایه هرم تست را تشکیل می‌دهند.
  • فصل 10 بر اساس فصل 9 ساخته شده است و نحوه نوشتن انواع دیگر تست‌ها در هرم تست، از جمله integration tests، consumer contract tests و component tests را شرح می‌دهد.
  • فصل 11 جنبه‌های مختلف توسعه services آماده تولید را پوشش می‌دهد، از جمله امنیت، الگوی پیکربندی خارجی و الگوهای مشاهده service. الگوهای مشاهده service شامل Log aggregation، Application metrics و Distributed tracing هستند.
  • فصل 12 الگوهای استقرار مختلفی را که می‌توانید برای استقرار services استفاده کنید، از جمله ماشین‌های مجازی، containers و serverless، شرح می‌دهد. همچنین
