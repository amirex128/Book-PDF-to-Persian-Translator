Scheduler—یک node را برای اجرای یک pod انتخاب می‌کند.

Controller manager—controllers را اجرا می‌کند، که اطمینان حاصل می‌کنند که state از the clus- ter با the intended state مطابقت دارد. For example, یک نوع از controller که به عنوان a replication controller شناخته می‌شود، تضمین می‌کند که تعداد مورد نظر از instances از یک service با راه‌اندازی و خاتمه دادن به instances، در حال اجرا هستند.

A node چندین component را اجرا می‌کند، شامل موارد زیر:

Kubelet—pods که در حال اجرا هستند را در node ایجاد و مدیریت می‌کند

Kube-proxy—Networking را مدیریت می‌کند، از جمله load balancing در سراسر pods
