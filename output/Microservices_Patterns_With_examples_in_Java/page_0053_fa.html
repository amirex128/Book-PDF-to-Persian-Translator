The Microservice architecture pattern language

مجموعه‌ای از الگوهای مرتبط از طریق این روابط، گاهی اوقات آنچه را که به عنوان یک زبان pattern شناخته می‌شود، تشکیل می‌دهند. الگوهای موجود در یک زبان pattern با هم کار می‌کنند تا مشکلات را در یک دامنه خاص حل کنند. به طور خاص، من زبان pattern معماری Microservice را ایجاد کرده‌ام. این مجموعه‌ای از معماری software و الگوهای طراحی به هم پیوسته برای microservices است. بیایید نگاهی به این زبان pattern بیندازیم.

1.6.3

Overview of the Microservice architecture pattern language

زبان pattern معماری Microservice مجموعه‌ای از الگوها است که به شما کمک می‌کند تا یک application را با استفاده از معماری microservice معماری کنید. شکل 1.10 ساختار سطح بالای زبان pattern را نشان می‌دهد. زبان pattern ابتدا به شما کمک می‌کند تا تصمیم بگیرید که آیا از معماری microservice استفاده کنید یا خیر. این معماری monolithic و معماری microservice را به همراه مزایا و معایب آن‌ها توصیف می‌کند. سپس، اگر معماری microservice برای application شما مناسب است، زبان pattern به شما کمک می‌کند تا با حل مسائل مختلف معماری و طراحی از آن به طور مؤثر استفاده کنید.

زبان pattern شامل چندین گروه از الگوها است. در سمت چپ در شکل 1.10 گروه الگوهای معماری application، الگوی معماری Monolithic و الگوی معماری Microservice قرار دارد. این‌ها الگوهایی هستند که ما در مورد آن‌ها بحث کرده‌ایم.

شکل 1.10

یک نمای سطح بالا از زبان pattern معماری Microservice که مناطق مختلف مشکلی را که الگوها حل می‌کنند، نشان می‌دهد. در سمت چپ الگوهای معماری application قرار دارند: معماری Monolithic و معماری Microservice. تمام گروه‌های دیگر الگوها مشکلاتی را حل می‌کنند که از انتخاب الگوی معماری Microservice ناشی می‌شود.
