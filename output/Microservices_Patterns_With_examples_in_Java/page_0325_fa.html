WRITING AUTOMATED TESTS

Automated tests معمولاً با استفاده از یک framework testing نوشته می‌شوند. JUnit، به عنوان مثال، یک فریم‌ورک testing محبوب جاوا است. شکل 9.2 ساختار یک automated test را نشان می‌دهد. هر تست توسط یک test method پیاده‌سازی می‌شود که متعلق به یک test class است.

یک automated test معمولاً از چهار فاز تشکیل شده است (http://xunitpatterns.com/Four%20Phase%20Test.html):

  1. Setup—Initialize کردن the test fixture، که شامل SUT و dependencies آن، به حالت اولیه مورد نظر است. به عنوان مثال، کلاس در حال تست را ایجاد کنید و آن را به حالتی که برای نشان دادن رفتار مورد نظرش لازم است، مقداردهی اولیه کنید.
  2. Exercise—فراخوانی SUT—به عنوان مثال، فراخوانی یک متد در کلاس در حال تست.
  3. Verify—ایجاد assertion هایی در مورد نتیجه فراخوانی و state SUT. به عنوان مثال، تأیید value بازگشتی متد و state جدید کلاس در حال تست.
