فصل 2

Decomposition strategies

به دلیل این تفکیک، تست business logic به صورت ایزوله بسیار آسان‌تر است. مزیت دیگر این است که معماری یک application مدرن را دقیق‌تر منعکس می‌کند. business logic را می‌توان از طریق آداپتورهای متعدد فراخوانی کرد، که هر کدام یک API یا UI خاص را پیاده‌سازی می‌کنند. business logic همچنین می‌تواند چندین آداپتور را فراخوانی کند، که هر کدام یک سیستم خارجی متفاوت را فراخوانی می‌کنند. معماری شش ضلعی یک راه عالی برای توصیف معماری هر service در یک معماری microservice است.

معماری لایه‌ای و شش ضلعی هر دو نمونه‌ای از سبک‌های معماری هستند. هر کدام بلوک‌های ساختمانی یک معماری را تعریف می‌کنند و محدودیت‌هایی را در روابط بین آن‌ها اعمال می‌کنند. معماری شش ضلعی و معماری لایه‌ای، به شکل یک معماری سه‌لایه، logical view را سازماندهی می‌کنند. بیایید اکنون معماری microservice را به عنوان یک سبک معماری تعریف کنیم که implementation view را سازماندهی می‌کند.

2.1.3

The microservice architecture is an architectural style

من در مورد مدل 4+1 و سبک‌های معماری بحث کرده‌ام، بنابراین اکنون می‌توانم معماری monolithic و microservice را تعریف کنم. هر دو سبک معماری هستند. معماری Monolithic یک سبک معماری است که implementation view را به عنوان یک مؤلفه واحد ساختار می‌دهد: یک فایل اجرایی یا WAR واحد. این تعریف چیزی در مورد views دیگر نمی‌گوید. به عنوان مثال، یک application monolithic می‌تواند یک logical view داشته باشد که بر اساس معماری شش ضلعی سازماندهی شده است.

معماری microservice نیز یک سبک معماری است. این implementation view را به عنوان مجموعه‌ای از مؤلفه‌های متعدد ساختار می‌دهد: فایل‌های اجرایی یا فایل‌های WAR. مؤلفه‌ها services هستند و کانکتورها، پروتکل‌های ارتباطی هستند که این services را قادر به همکاری می‌کنند. هر service معماری logical view خود را دارد، که معمولاً یک معماری شش ضلعی است. شکل 2.3 یک معماری microservice ممکن برای application FTGO را نشان می‌دهد. services در این معماری با قابلیت‌های تجاری، مانند مدیریت سفارش و مدیریت رستوران، مطابقت دارند.

در ادامه این فصل، منظور از قابلیت تجاری را شرح می‌دهم. کانکتورها بین services با استفاده از مکانیزم‌های ارتباط بین فرآیندی مانند REST APIs و پیام‌رسانی ناهمزمان پیاده‌سازی می‌شوند. فصل 3 در مورد ارتباط بین فرآیندی با جزئیات بیشتری بحث می‌کند.
