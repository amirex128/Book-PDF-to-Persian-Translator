معماری نرم‌افزار یک سیستم محاسباتی، مجموعه‌ای از ساختارها است که برای استدلال در مورد سیستم مورد نیاز است، که شامل عناصر نرم‌افزاری، روابط بین آن‌ها و خواص هر دو است.

THE 4+1 VIEW MODEL OF SOFTWARE ARCHITECTURE

به طور ملموس‌تر، معماری یک application را می‌توان از دیدگاه‌های متعددی مشاهده کرد، به همان روشی که معماری یک ساختمان را می‌توان از دیدگاه‌های ساختاری، لوله‌کشی، برق و سایر دیدگاه‌ها مشاهده کرد. Phillip Krutchen یک مقاله کلاسیک با عنوان “Architectural Blueprints—The ‘4+1’ View Model of Software Architecture” (www.cs.ubc.ca/~gregor/teaching/papers/4+1view-architecture.pdf)، که مدل 4+1 را توصیف می‌کند، نوشت. مدل 4+1، که در شکل 2.1 نشان داده شده است، چهار دیدگاه مختلف از یک معماری software را تعریف می‌کند. هر کدام جنبه خاصی از معماری را توصیف می‌کنند و از یک مجموعه خاص از عناصر software و روابط بین آن‌ها تشکیل شده‌اند.

هدف از هر view به شرح زیر است:

  • Logical view—عناصر software که توسط developers ایجاد شده‌اند. در زبان‌های شی گرا، این عناصر کلاس‌ها و بسته‌ها هستند. روابط بین آن‌ها، روابط بین کلاس‌ها و بسته‌ها، از جمله وراثت، انجمن‌ها و depends-on هستند.
  • Implementation view—خروجی سیستم build. این view شامل ماژول‌ها است که نشان دهنده کد بسته‌بندی شده و مؤلفه‌ها هستند که قابل اجرا هستند
