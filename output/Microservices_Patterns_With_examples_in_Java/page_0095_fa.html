Interprocess communication در یک معماری microservice

Mary و تیمش، مانند اکثر توسعه‌دهندگان دیگر، تجربه‌ای در مورد مکانیسم‌های inter-process communication (IPC) داشتند. اپلیکیشن FTGO دارای یک REST API است که توسط اپلیکیشن‌های موبایل و JavaScript سمت مرورگر استفاده می‌شود. همچنین از موارد مختلفی استفاده می‌کند.

این فصل موارد زیر را پوشش می‌دهد:

  • اعمال الگوهای communication: Remote procedure invocation، Circuit breaker، Client-side discovery، Self registration، Server-side discovery، Third party registration، Asynchronous messaging، Transactional outbox، Transaction log tailing، Polling publisher
  • اهمیت interprocess communication در یک معماری microservice
  • تعریف و تکامل APIs
  • گزینه‌های مختلف interprocess communication و مقایسه آن‌ها
  • مزایای سرویس‌هایی که با استفاده از asynchronous messaging ارتباط برقرار می‌کنند
  • ارسال قابل اطمینان پیام‌ها به عنوان بخشی از یک database transaction
