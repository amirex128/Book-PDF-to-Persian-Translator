CHAPTER 7

Implementing queries in a microservice architecture

مانند بسیاری از application‌های دیگر، یک REST API دارد. clients آن شامل applicationهای تلفن همراه FTGO، JavaScript در حال اجرا در مرورگر و applicationهای توسعه یافته توسط شرکا است. در چنین معماری monolithic، APIای که به clients ارائه می‌شود، API monolith است. اما هنگامی که تیم FTGO شروع به استقرار microservices می‌کند، دیگر یک API وجود ندارد، زیرا هر service، API خود را دارد. Mary و تیمش باید تصمیم بگیرند که application FTGO اکنون باید چه نوع API را به clients خود ارائه دهد. به عنوان مثال، آیا clients باید از وجود services آگاه باشند و مستقیماً به آن‌ها درخواست ارسال کنند؟

این فصل موارد زیر را پوشش می‌دهد

  • چالش طراحی APIs که از مجموعه‌ای متنوع از clients پشتیبانی می‌کنند
  • اعمال الگوی API gateway و Backends for frontends
  • طراحی و پیاده‌سازی یک API gateway
  • استفاده از برنامه‌نویسی واکنشی برای ساده‌سازی ترکیب API
  • پیاده‌سازی یک API gateway با استفاده از GraphQL
